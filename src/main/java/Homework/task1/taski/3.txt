Техническое задание для разработки программного модуля "Транспортная система" на Java

1. Цель проекта:
Разработка набора классов и интерфейсов в Java для моделирования
транспортной системы города. Система должна включать различные типы транспортных средств и их управление.

2. Описание системы:

Создание классов Vehicle, Bus, Taxi, Tram, TransportManager и интерфейсов TransportControl, Maintenance.
Каждый класс должен содержать минимум 4 поля различных типов.
Для хранения данных о транспортных средствах использовать массивы.
Реализация бизнес-логики в методах классов.
3. Требования к классам и интерфейсам:

3.1. Абстрактный класс Vehicle:
Поля: String model, int capacity, double fare, String route.
Абстрактные методы для основных операций транспортного средства.

3.2. Класс Bus:
Наследуется от Vehicle.
Дополнительные поля: boolean accessibility, int busNumber.
Реализация специфических методов для автобуса.

3.3. Класс Taxi:
Наследуется от Vehicle.
Дополнительные поля: String licensePlate, boolean available.
Реализация специфических методов для такси.

3.4. Класс Tram:
Наследуется от Vehicle.
Дополнительные поля: int trackWidth, boolean electric.
Реализация специфических методов для трамвая.

3.5. Класс TransportManager:
Поля: Vehicle[] vehicles, int totalVehicles, String managerName.
Реализация методов управления транспортными средствами.

3.6. Интерфейс TransportControl:
Методы для управления движением и маршрутами транспортных средств.

3.7. Интерфейс Maintenance:
Методы для обслуживания и ремонта транспортных средств.

4. Бизнес-логика:
Определение логики управления транспортными средствами, маршрутами и обслуживанием.
Взаимодействие между классами для обеспечения эффективной работы транспортной системы.
